## üß† What I Learned

Deployment is about keeping systems alive, not just running commands  
A VM is a stateful machine where memory resets but disk persists  
SSH is not deployment; long-lived systems must survive disconnects  

Containers are just Linux processes wrapped by Docker  
A container exits when its main process exits, even with exit code 0  
Detached mode does not imply persistence or uptime  

Logs are the only reliable source of truth during deployment and failure  
Exit codes communicate how and why processes terminated  
Manual recovery must be possible before automation is introduced  

Docker does not guarantee uptime by default  
Running containers do not survive Docker daemon restarts  
Runtime state does not survive VM reboots  

Disk state (images, container definitions, repositories) is the foundation of recovery  
Deployment confidence comes from rebuilding runtime state repeatedly  

Rebuild-from-memory is the true test of deployment understanding  
Calm inspection and deliberate recovery matter more than automation
