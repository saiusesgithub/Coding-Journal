## üß† What I Learned

Docker Compose orchestrates containers; it does not replace Docker
Dockerfile builds images; Compose runs containers
A service is a container runtime definition
Service name = DNS hostname inside the Compose network

`localhost` inside a container refers to the container itself
Ports are for host ‚Üî container, not container ‚Üî container
Containers communicate via Docker network, not exposed ports

ENV variables control runtime behavior
Changing ENV does not require image rebuild
Changing application code requires image rebuild

Empty or mis-indented YAML blocks cause Compose validation errors
Config errors and runtime errors are different and must be debugged differently

`depends_on` ‚â† readiness
Container start ‚â† service ready
Startup order does not guarantee dependency availability

False-positive health checks are dangerous
Green logs do not guarantee a correct system

Containers are parts of a system, not isolated tools
